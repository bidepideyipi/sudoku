# 数独题库系统 PRD

## 一、产品定位

### 1.1 产品概述

构建一个自动化数独题库生成与管理系统，为数独游戏应用提供持续、稳定的题目供应能力。系统支持多难度分级，确保题目质量（唯一解、可解性），并可根据业务需求灵活扩展题库规模。

### 1.2 目标用户

- **主要用户**：数独游戏玩家（覆盖新手到专家各水平段）
- **次要用户**：运营人员（需要题库管理和配置能力）

### 1.3 核心价值

- **自动化供应**：无需人工出题，系统可持续生成新题
- **难度精准**：多维度难度评估，匹配不同玩家水平
- **质量保证**：每题必经唯一解验证，杜绝无效题目
- **可扩展性**：支持百万级题库存储和高效查询

---

## 二、功能范围

### 2.1 核心功能

| 功能模块 | 功能描述 | 优先级 |
|---------|---------|--------|
| **终盘生成** | 生成符合数独规则的完整9×9数字终盘 | P0 |
| **题目挖空** | 按策略挖空数字，生成 playable 题目 | P0 |
| **唯一解验证** | 确保挖空后的题目有且仅有一个解 | P0 |
| **难度分级** | 对题目进行多级难度分类（简单/中等/困难/专家） | P0 |
| **题库入库** | 将生成的题目持久化存储，支持批量导入 | P1 |
| **题目查询** | 按难度、未使用等条件检索题目 | P1 |

### 2.2 暂不包含

- 非标准9×9数独变体（如6×6、12×12、对角线数独等）
- 用户自定义题目编辑器
- 题目推荐算法（基于用户历史表现）
- 题目社交功能（分享、点赞、评论）

---

## 三、难度分级定义

### 3.1 难度等级

| 等级 | 名称 | 目标用户 | 特征 |
|------|------|---------|------|
| Level 1 | 简单 | 新手入门 | 挖空25个，较多唯一候选线索 |
| Level 2 | 中等 | 有经验玩家 | 挖空35个，需基础推理技巧 |
| Level 3 | 困难 | 熟练玩家 | 挖空45个，需中级推理技巧 |
| Level 4 | 专家 | 高手玩家 | 挖空55个，可能需高级技巧 |

### 3.2 难度度量标准

采用**多指标综合评分**方式：

- **挖空数量**：基础分值
- **求解步数**：标准求解器所需步数
- **候选数复杂度**：初期每个空格的平均候选数
- **逻辑技巧需求**：是否需要排除法、X-Wing 等高级技巧

## 四、实现路径

### 4.1 第一阶段：核心生成能力（MVP）

**目标**：实现基础的数独题目生成功能

1. 实现回溯算法，生成完整数独终盘
2. 实现挖空逻辑，按固定数量挖空
3. 实现唯一解验证，确保题目有效性
4. 输出基础的题目数据结构

**交付标准**：能够持续生成有效的数独题目

### 4.2 第二阶段：难度分级系统

**目标**：实现多难度题目分类

1. 实现标准数独求解器，用于难度评估
2. 建立难度评分模型，综合多指标计算难度值
3. 将题目按4个等级分类入库
4. 支持按难度检索题目

**交付标准**：生成的题目可按难度正确分类，难度分布合理

### 4.3 第三阶段：题库管理与优化

**目标**：支持大规模题库存储和高效查询

1. 设计题库存储方案（本地/云端）
2. 实现批量预生成和入库流程
3. 建立题目使用状态追踪（已使用/未使用）
4. 实现题目去重机制

**交付标准**：支持万级以上题库，查询响应时间<100ms

### 4.4 第四阶段：质量与体验优化（可选）

**目标**：提升题目质量和生成效率

1. 实现对称挖空选项，提升题目美观度
2. 优化生成算法效率，降低耗时
3. 建立题目质量评估指标（如解题路径多样性）
4. 支持运营后台的题库管理界面

## 五、非功能性需求

### 5.1 性能

- 单题目生成时间：< 500ms（中等难度）
- 题库查询响应：< 100ms

### 5.2 可靠性

- 题目有效性：100%（唯一解保证）
- 题目不重复率：> 99.9%

### 5.3 可扩展性

- 支持从本地存储扩展到云端存储
- 支持后续添加新的难度等级
- 算法模块可独立升级优化

## 六、技术架构选型

### 6.1 存储架构

采用**服务端 + 本地 SQLite 缓存**的混合架构：

| 组件 | 职责 |
|------|------|
| 服务端题库 | 题目主存储、生成、管理，支持百万级题库 |
| 同步 API | 负责 APP 与服务端之间的题目数据同步 |
| 本地 SQLite | 本地题目缓存，支持离线游戏 |

### 6.2 架构优势

- **题库规模无限制**：服务端可管理百万级题目
- **支持热更新**：题库更新无需发版，服务端下发即可
- **离线可用**：同步后本地 SQLite 支持完全离线游戏
- **数据可收集**：可收集用户解题行为，用于优化难度评级

### 6.3 数据同步流程

```
APP 启动 → 检查本地存量 → 存量不足/请求更新 → 从服务端拉取 → 写入 SQLite
```
