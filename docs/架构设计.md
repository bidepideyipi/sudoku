# 数独题库系统架构设计

## 一、系统概述

数独题库系统采用**服务端-客户端分离架构**，支持大规模题库管理和多端访问。系统由服务端和客户端（Flutter APP）两部分组成，通过同步 API 进行数据交互。

---

## 二、组件划分

### 2.1 服务端组件

| 组件 | 职责 |
|------|------|
| 题库生成器 | 负责数独终盘生成、挖空、唯一解验证、难度分级 |
| 题库存储 | 持久化存储题目数据，支持百万级规模 |
| 同步 API | 对外提供题目同步接口，支持批量拉取和增量更新 |
| 题库管理服务 | 题目去重、使用状态追踪、质量管理 |

### 2.2 客户端组件（Flutter APP）

| 组件 | 职责 |
|------|------|
| SQLite 本地库 | 本地题目缓存存储，支持离线访问 |
| 同步管理器 | 负责与服务端同步题目数据 |
| 题目查询服务 | 从本地库查询题目，供游戏界面使用 |
| 游戏界面 | 数独游戏 UI，展示题目和处理用户交互 |

---

## 三、组件关系

### 3.1 数据流向

```
┌─────────────────┐
│  服务端题库生成器  │
└────────┬─────────┘
         │ 生成并入库
         ▼
┌─────────────────┐
│   服务端题库存储   │
└────────┬─────────┘
         │ 同步 API
         ▼
┌─────────────────┐     ┌─────────────────┐
│ 客户端同步管理器   │ ──▶ │ 客户端 SQLite库  │
└────────┬─────────┘     └────────┬─────────┘
         │                        │
         ▼                        ▼
┌─────────────────┐     ┌─────────────────┐
│  游戏界面 UI     │ ◀─ │  题目查询服务    │
└─────────────────┘     └─────────────────┘
```

### 3.2 组件交互说明

| 交互 | 描述 |
|------|------|
| 生成器 → 题库存储 | 生成的新题批量写入服务端题库 |
| 题库存储 → 同步 API | API 从题库检索题目并返回 |
| 同步管理器 ← 同步 API | 客户端拉取题目数据 |
| 同步管理器 → SQLite | 将拉取的题目写入本地库 |
| 题目查询服务 → SQLite | 按条件查询本地题目 |
| 游戏界面 → 题目查询服务 | 请求获取题目展示 |

## 四、系统属性目标

### 4.1 可用性
- 客户端支持完全离线游戏（同步后）
- 服务端可用性目标 99.9%

### 4.2 性能
- 同步 API 响应时间：< 200ms（单批次）
- 本地题目查询：< 50ms
- 单次同步可拉取 100-500 题目

### 4.3 可扩展性
- 服务端题库可扩展至千万级
- 支持多端访问（未来可扩展 Web 端、小程序等）
- 算法模块可独立升级

### 4.4 一致性
- 题目数据以服务端为源头
- 本地库与服务端采用最终一致性模型
- 支持增量更新，减少数据传输量
